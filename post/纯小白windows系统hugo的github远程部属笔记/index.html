Hugo简介

  • Hugo 是 Go 编写的静态网站生成器，速度快，易用，可配置。Hugo 有一个内容和模板目录，把他们渲染到完全的 HTML 网站。
  • Hugo 依赖于 Markdown 文件，元数据字体 。用户可以从任意的目录中运行 Hugo，支持共享主机和其他系统。
  • Hugo 只需要几分之一秒就可以渲染一个经典的中型网站，最好网站的每个部分渲染只需 1 毫秒。
  • Hugo 非常适合博客，文档等等网站的生成。
  • Hugo 当前提供 Windows，Linux，FreeBSD，NetBSD 和 OS X (Darwin) 的 x64, i386 和 ARM 架构的二进制预构建包。
  • Hugo 的源代码可以通过 Go 编译器工具链编译到任意地方运行，例如： DragonFly BSD, OpenBSD, Plan 9 和 Solaris。

安装hugo

hugo下载地址

我目前选的版本是hugo_0.55.6_Windows-64bit.zip，随便解压到一个位置，然后在系统环境变量添加文件位置，我设置的是D:\Program Files\hugo_0.55.6_Windows-64bit。

生成站点

创建部属的文件夹，会在当前位置自动生成一个文件夹。

PS D:\Program Files> hugo new site MyBlog

创建的站点文件目录说明：

  • archetypes ：存放default.md，头文件格式

  • content ：一般在content/post目录存放博客文章（.md文件）

  • data ：存放自定义模版，导入的toml文件（或json，yaml）

  • layouts ：layouts目录存放的是网站的模板文件

  • static ：static目录存放js/css/img等静态资源

  • config.toml ：网站的配置文件

下载hugo主题

hugo主题地址

随意挑个喜欢的主题，我选择的是m10c主题。

打开m10c主题，复制里面的git代码，直接用git下载到刚生成的themes文件夹（别忘了powershell要先进入到themes文件夹），

PS D:\Program Files\MyBlog\themes> git clone https://github.com/vaga/hugo-theme-m10c.git themes/m10c

然后查看一下themes文件夹下是否下载好主题了。

PS D:\Program Files\MyBlog\themes> tree




本地部属预览调试

创建博客文件

对于博客.md文件，通常放在 content/post 目录中，

PS D:\Program Files\MyBlog>hugo new post/MyFirstBlog.md

然后在MyFirstBlog.md文件里输出一些内容。

---
title: "那刻温存丶的博客"
date: 2019-06-19T14:13:20+08:00
draft: false
---
# 感谢

谢谢**那刻温存**小白同学的细心写作。

备注：“—”之间的内容是自动生成的，把draft值改为false，原因见下面的启动 hugo 自带的服务器调试预览博客。

启动 hugo 自带的服务器调试预览博客

一定要在项目根目录（我的是D:\Program Files\MyBlog）下，通过 hugo server 命令可以使用 hugo 内置服务器调试预览博客。--theme 选项可以指定主题，--watch 选项可以在修改文件后自动刷新浏览器，--buildDrafts 包括标记为草稿（draft）的内容。不包括的话就不用写--buildDrafts，这样就要把需要显示的MyFirstBlog.md文件中的draft: true改为false。

powershell里输入hugo server --theme=m10c --watch(注意要在根目录)，我没用--buildDrafts参数

PS D:\Program Files\MyBlog> hugo server --theme=m10c --watch

如果成功会显示一些信息，从里面找到http://localhost:1313/，复制下来，打开浏览器输入刚复制的本地地址。如果关闭了powershell，浏览器就看不到了。




github远程部属

生成静态网站

PS D:\Program Files\MyBlog> hugo --theme=m10c --baseUrl="https://teyying.github.io/"

所有静态页面都会生成到 public 目录下。

创建仓库

我的仓库名是：teyying.github.io。创建仓库时的”/“后面,teyying必须是你的用户名，要和”/“前面的名字一样。

推送文件到github仓库

在public目录位置，依次输入以下git语句（“https://github.com/teyying/teyying.github.io.git”）

要写成你自己的地址。第四句添加远程仓库时，根据提示输入你github的账号和密码。

PS D:\Program Files\MyBlog\public> git init
PS D:\Program Files\MyBlog\public> git add .
PS D:\Program Files\MyBlog\public> git commit -m "我的hugo博客学习"
PS D:\Program Files\MyBlog\public> git remote add origin https://github.com/teyying/teyying.github.io.git
PS D:\Program Files\MyBlog\public> git push -u origin master

最后呈现

打开浏览器，“https://github.com/teyying/teyying.github.io.git”，只输入粗体字部分https://teyying.github.io。

修改配置

打开config.toml文件

修改以下内容（目前我只会这些参数）

baseURL = "http://example.org/"
languageCode = "en-us"
title = "那刻温存丶的博客"

[params]
    author = "那刻温存丶"
    description = "看破了，是感慨，看淡了，是境界..."
    avatar = "01.jpg"  # 改变头像，要把图片放在你的主题文件夹下的static文件夹中

[[params.social]]
    name = "github"
    url = "https://github.com/vaga"

改变头像

要把图片放在你的主题文件夹下的static文件夹中

博客插入图片

同改变头像一样，，可以static文件夹下新建一个文件夹，比如我放在D:\Program Files\MyBlog\themes\m10c\static\纯小白windows系统hugo的github远程部属笔记\01.jpg，然后博客文档（.md）里在适当位置添加图片就OK了。

提示：

当以上都成功后，发表新博客文章及修改内容时，首先都要在根目录D:\Program Files\MyBlog再生成一次静态文件

PS D:\Program Files\MyBlog> hugo --theme=m10c --baseUrl="https://teyying.github.io/"

然后在D:\Program Files\MyBlog\public目录下依次执行以下git语句

PS D:\Program Files\MyBlog\public> git add .
PS D:\Program Files\MyBlog\public> git commit -m "修改"
PS D:\Program Files\MyBlog\public> git push

不需要再次初始化git（git init），和添加远程仓库了git remote add origin https://github.com/teyying/teyying.github.io.git，推送可以直接git push了。
